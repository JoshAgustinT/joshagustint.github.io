DRAMBlast

Project focused on the design of DRAMBlast, a next-generation in-memory hash table that reaches the limits of modern hardware bandwidth (Or, as we like to call it: the fastest hash table in the world). This work explores the interaction between CPU execution resources, memory bandwidth, and prefetch behavior to develop practical design principles for ultra-fast hash tables. DRAMBlast achieves 3200–5000 million operations per second (Mops) for lookups and 2150–2800 Mops for insertions on a modern two-socket server, reaching performance bounded only by the memory subsystem itself. Key innovations include a multi-level prefetching scheme, a compute and memory aware table layout, and a conflict resolution strategy optimized for memory bandwidth utilization. Contributed as a co-author with MARS Research Group.
Submitted to EuroSys 2026.

IPC

This project explores hardware–software co-design principles for practical fine-grained inter-process isolation. It proposes a set of design principles and a gem5 based prototype of a hardware isolation scheme that achieves strong isolation with minimal hardware extensions and performance overhead. The goal is to enable efficient, low-cost isolation within complex operating system kernels by leveraging lightweight architectural support. Research being conducted with MARS Research Group.
